## 一、项目定位

LLM-merge 是运行在 Cloudflare Workers 或 Pages Advanced Mode 上的单文件 LLM 聚合网关。它将多个上游模型服务统一为 OpenAI 兼容的 `/v1` Base URL，并提供中文管理后台。

## 二、已完成

### 1. 部署与配置

- 支持 Cloudflare Workers 直接部署。
- 支持 Cloudflare Pages Advanced Mode，以 `_worker.js` 作为入口，无需构建步骤。
- `wrangler.toml` 保持为通用本地开发配置，不写死 Pages 生产构建目录。
- 生产环境变量与机密可以在 Cloudflare Workers / Pages 仪表板中配置。
- 支持 `ADMIN_TOKEN`、`API_KEY_CRYPT_SECRET`、`REQUEST_TIMEOUT_MS`、`STREAM_IDLE_TIMEOUT_MS`、`UPSTREAM_COOLDOWN_TTL`、`MODEL_CACHE_TTL`、`UPSTREAMS_JSON`、`CLIENTS_JSON` 等配置。
- 支持 KV binding `KV`，用于保存网关配置、客户端 Key、上游配置、提示词、上下文、模型缓存和冷却状态。
- 支持 Analytics Engine 绑定 `ANALYTICS`，用于历史日志和统计查询；未配置查询能力时保留 KV 或内存 fallback。

### 2. 管理后台与前端交互

- 管理后台路径由 `ADMIN_TOKEN` 控制，支持大小写不敏感以及连字符/下划线形式的宽松匹配。
- 管理后台主要界面已改为中文，源码使用 Unicode 转义避免部署环境编码导致乱码。
- 已修复后台按钮初始化顺序问题，按钮可正常绑定事件并执行请求。
- 支持添加、编辑、启用/停用、删除和保存上游。
- 支持上游配置导入/导出、模型导入、健康检查和模型测速。
- 支持按上游分组展示，并显示健康状态、活跃状态和能力信息。
- 支持模型选择器、来源分组、标签筛选、上下文长度备注和批量应用。
- 支持生成客户端 Key、复制 Key/JSON、刷新列表和删除客户端。
- 支持系统提示词、全局上下文、上下文片段、客户端范围选择、按需注入、导入/导出和大文本拆分。
- 支持请求统计图表、Token 统计、最近请求日志、日志筛选和活跃上游信息。
- 已拆分 `admin-page.js`，减少 `_worker.js` 中的页面模板负担。

### 3. 网关协议与模型能力

- OpenAI 兼容：`/v1/models`、`/v1/chat/completions`、`/v1/embeddings`。
- Responses API 兼容层：`/v1/responses`、`/v1/responses/compact`。
- Anthropic / Claude 风格入口：`/v1/messages`。
- 支持模型别名和来源别名，并在公开模型列表与响应中保持一致。
- 支持客户端模型权限、会话级模型锁定和模型路由校验。
- 支持 GLM、Qwen、MiniMax、Kimi、DeepSeek、Nemotron、Mistral 等 NIM 或官方协议桥接。
- 支持 OpenRouter、DeepInfra、Together AI、DeepSeek、Kimi、MiniMax、Cloudflare Workers AI REST 和自定义 OpenAI 兼容上游。

### 4. 路由与稳定性

- 支持故障转移、按权重负载均衡、优先级排序、延迟排序和活跃连接排序。
- 支持 Hedged Request 和 Gateway Fast 模式，用于降低首包等待时间。
- 支持上游失败冷却、重试和 524 超时错误处理。
- 支持流式响应保活、空闲超时、SSE 结束处理和取消后的清理。
- 对 Anthropic 流式代理、HTML 错误页、上游应用层错误和 Cloudflare 502/524 场景做了保护。
- 日志记录请求、模型、上游、状态、Token、工具调用、trace id 和流式诊断信息。

## 四、已知注意事项

- `ADMIN_TOKEN` 主要用于隐藏管理路径，不等同于完整的登录系统；生产环境应使用足够随机的机密值。
- 管理后台依赖名为 `KV` 的绑定；未绑定 KV 时，管理 API 会返回配置错误。
- `API_KEY_CRYPT_SECRET` 变化后，历史加密上游 Key 可能无法解密，生产环境应保持稳定。
- 上游配置导出包含明文 API Key，导出文件需要妥善保存。
- Analytics Engine 查询需要绑定和具备 Account Analytics Read 权限的 API Token。
- 内存实时统计可能随 Worker isolate 回收而丢失，历史统计应以 Analytics Engine 为准。
- 当前没有浏览器自动化回归测试，真实 Pages 域名仍需做一次部署后的页面点击和接口冒烟验证。
- 工作区中已有几份未跟踪文本文件，本次进度文件没有将它们纳入提交。

## 五、下一步建议

1. 在 Cloudflare Pages 项目中确认 `KV`、`ANALYTICS` 绑定。
2. 在 Variables and Secrets 中配置 `ADMIN_TOKEN`、`API_KEY_CRYPT_SECRET` 和实际上游配置。
3. 访问 `https://你的域名/{ADMIN_TOKEN}`，验证中文页面、配置保存、上游健康检查、测速、客户端 Key 和日志统计。
4. 使用 OpenAI SDK、Cherry Studio 或 Anthropic 客户端分别验证 `/v1/chat/completions`、`/v1/responses` 和 `/v1/messages`。
5. 对实际上游进行一次流式请求和失败切换测试，确认超时、重试和冷却参数符合生产需求。
